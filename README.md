# Git Tutorial

A practical Git and GitHub workflow tutorial repository focused on collaborative development practices. This project is used as a training and reference guide for version control, team branching strategies, pull requests, rebasing, and code review workflows.

## Fork Context

- This repository is a **fork** of [`RealEskalate/git-tutorial`](https://github.com/RealEskalate/git-tutorial).
- The current branch is reported as **up to date** with the upstream `main` branch.
- The fork is used for guided collaboration and exercise-based Git practice.

## Contents

- **`docs/git.md`**: Main Git workflow guide covering fundamentals through advanced team collaboration practices.
- **`docs/support/`**: Supporting visual assets and diagrams for Git concepts (branching, merge, rebase, PR flow, etc.).

## What This Tutorial Covers

- **Version control fundamentals**
  - Git, GitHub, repositories, staging, commits, push/pull workflows

- **Team collaboration workflow**
  - Branch creation and naming conventions
  - Merge and rebase strategies
  - Conflict handling and resolution

- **Pull request best practices**
  - PR creation flow
  - Conventional commit style guidance
  - Small PR principles and review etiquette

- **Code review navigation**
  - How to evaluate PR quality
  - What to check in reviews
  - Constructive comment style and severity labels

- **Practical exercises**
  - Fork-and-collaborate workflow
  - Rebase-before-merge flow
  - Squash-and-merge usage in team settings

## Usage

1. Clone the fork:
   ```sh
   git clone https://github.com/natnael-eyuel-dev/git-tutorial.git
   ```
2. Open the guide:
   - Read `docs/git.md` from top to bottom.
3. Practice the workflow with teammates:
   - Create branches
   - Push changes
   - Open pull requests
   - Review and merge using the documented process

## Intended Audience

- Students and junior developers learning Git team workflows
- Bootcamp/project teams adopting a consistent contribution process
- Anyone needing a practical Git branching + PR handbook
